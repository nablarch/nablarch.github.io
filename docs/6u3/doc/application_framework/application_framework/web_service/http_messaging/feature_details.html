重要

排他制御 機能は、クライアント(taglib)との連動が前提であるため、 HTTPメッセージングでは使用できない。

URIとアクションクラスのマッピング¶

  • HTTPリクエストディスパッチハンドラ

補足

HTTPメッセージングでは ルーティングアダプタ を使用できない。 HTTPメッセージングは、MOMメッセージング が提供する MessagingAction でアクションクラスを作成するため、URIに応じてアクションクラスのメソッドを呼び分ける想定がないため。
